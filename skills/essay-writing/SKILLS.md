# SKILL: Produce a concise, causal mini-essay (generic)

## Purpose
Create a short piece of writing that communicates an idea quickly and precisely. The piece must be causally disciplined: every sentence must earn its place by being a cause or effect of the sentence before it.

## Core Writing Principles (non-negotiable)

### 1. Causal Discipline
Every sentence in an explaining paragraph must advance a causal chain. The test: ask "is this sentence a cause or effect of the sentence before it?" If not, cut it. Being true is not enough to earn a place. Being relevant is not enough. It must be causally necessary.

Never include decorative context — background, history, or scene-setting — unless that prior state is the direct cause of the shift being described.

### 2. Mechanism Before Description
Every explaining paragraph opens by naming the mechanism, not describing the situation. Format: "<Subject> does <thing> by <mechanism>." Description follows only if it advances the causal chain.

### 3. Concrete Causality
Name the bottleneck, constraint, or point of leverage explicitly. Use concrete nouns — "constraint," "bottleneck," "closure," "specification," "convergence," "tradeoff" — and verbs that show causality — "therefore," "because," "so," "which means," "this shifts," "produces," "dissolves." Avoid jargon unless defined in-line.

### 4. No Filler
No emojis. No bullet points or lists unless explicitly requested. No generic acknowledgments. No hedging phrases. Start immediately. Tone: direct, precise, non-fluffy, non-preachy.

### 5. Respect User Phrasing
When the user provides a specific sentence or phrasing, use it exactly without paraphrasing, unless asked to revise it.

## Structure Is a Variable — Ask First

Structure (number of paragraphs, presence of a headline, use of a contemplative lens, etc.) is determined per session by asking the user. Do not assume a default structure. Before drafting, ask:

- **Headline?** Should there be one? If yes, what tone — punchy, neutral, provocative?
- **How many paragraphs?** What should each one do (state the claim, explain the cause, explain the problem, offer a solution, etc.)?
- **Contemplative or philosophical lens?** If yes, which tradition or keep secular?
- **Any specific sentences or phrasings** the user wants preserved verbatim?
- **Word count or length constraint?** If none given, default to tight and short.

Only proceed to draft once the structure is confirmed.

## Self-Correction Rules (apply during revision)
- If a sentence in an explaining paragraph does not advance the causal chain, remove it — do not soften or reposition it.
- If the opening of a paragraph is descriptive rather than mechanistic, rewrite it to name the mechanism first.
- If the user flags a sentence as irrelevant, the correct response is to cut it, not defend or reframe it.
- If a paragraph has been revised more than once for the same issue, re-examine the opening sentence — the root cause is usually there.
